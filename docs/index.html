Data Integrity & Validation

  • Price Logic: Zero or negative prices are treated as critical business errors (TC-INV-002).
  • ID Format: Cart IDs are expected to be unique strings; numeric-only IDs are invalid (TC-CART-005).
  • Sanitization: System should trim whitespaces while supporting Unicode/Emojis (TC-INV-007).

Business Logic & Discounts

  • Rounding: Total amounts are rounded down (floor) in favor of the customer (TC-DISC-001).
  • Persistence: Active discounts must persist and recalculate when new items are added (TC-DISC-003).
  • Empty States: Applying promo codes to an empty cart is considered a validation error.

Technical Constraints

  • UI Sync: Frontend values must strictly match API responses (TC-UI-004).
  • Load Limits: API is assumed to handle up to 200 sequential item additions (TC-STRESS-001).
  • State: Cart data must persist across page reloads (F5), currently tracked as BUG-011.
